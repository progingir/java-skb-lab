Проект реализует rest api с ограничением количества запросов с использованием aop

## Что имеется
- Аннотация `@RequestLimit` для методов API
- Подсчет количества запросов в заданном временном окне
- Блокировка запросов при превышении лимита с возвратом ошибки HTTP 429
- Конфигурация лимитов через `application.properties`
- Логирование ограничений через исключения

## Как работает код

### Архитектура
1. **LimitConfig**:
    - Читает настройки лимитов из `application.properties` (`request.limiter.max-requests`, `request.limiter.time-window-seconds`)
    - Определяет максимальное количество запросов и временное окно

2. **ApiController**:
    - REST контроллер с эндпоинтом:
        - `GET /api/test/run` — тестовый метод с аннотацией `@RequestLimit`

3. **RequestStore**:
    - Хранит временные метки запросов в `ArrayList`
    - Проверяет, не превышен ли лимит запросов в заданном временном окне
    - Удаляет устаревшие запросы

4. **LimitAspect**:
    - Аспект с аннотацией `@Aspect`
    - Перехватывает методы с `@RequestLimit` и проверяет лимит через `RequestStore`
    - Выбрасывает `LimitExceededException` при превышении лимита

5. **ExceptionHandlerController**:
    - Обрабатывает `LimitExceededException`
    - Возвращает JSON с ошибкой (код 429, сообщение, путь, время)

### Логика работы

#### Вызов API-метода (GET /api/test/run)
1. Аспект `LimitAspect` перехватывает вызов метода
2. `RequestStore` проверяет количество запросов в окне времени (например, 5 запросов за 60 секунд)
3. Если лимит не превышен:
   - Записывается новая временная метка
   - Метод выполняется, возвращается "Hello, World!"
4. Если лимит превышен:
   - Выбрасывается `LimitExceededException`
   - Возвращается ошибка HTTP 429 с деталями
5. Все шаги логируются через обработчик исключений
