Проект реализует REST API для управления напоминаниями с использованием событий и слушателей

## Что имеется
- Три типа слушателей событий: обычный, асинхронный и транзакционный
- Публикация событий для каждого слушателя
- Транзакционный слушатель обрабатывает события только в транзакции
- Используется H2 in-memory база данных
- Логирование всех этапов публикации и обработки событий

## Как работает код

### Архитектура
1. **AsyncConfiguration**:
    - Включает поддержку асинхронной обработки через `@EnableAsync`
    - Позволяет выполнять асинхронные слушатели событий

2. **ReminderApi**:
    - REST контроллер с эндпоинтами:
        - `POST /api/reminders` — создание напоминания
        - `PUT /api/reminders/{id}` — обновление напоминания

3. **ReminderService**:
    - Содержит бизнес-логику для создания и обновления напоминаний
    - Публикует три типа событий: `ReminderReceivedEvent`, `ReminderCreatedEvent`, `ReminderNotifiedEvent`

4. **ReminderEventListener**:
    - Обрабатывает события:
        - Обычный слушатель для `ReminderReceivedEvent`
        - Транзакционный слушатель (`@TransactionalEventListener`) для `ReminderCreatedEvent`
        - Асинхронный слушатель (`@Async`) для `ReminderNotifiedEvent`

5. **События**:
    - `ReminderReceivedEvent` — логирует получение текста напоминания
    - `ReminderCreatedEvent` — добавляет "!" к тексту напоминания (только в транзакции)
    - `ReminderNotifiedEvent` — имитирует отправку уведомления с задержкой 2 секунды

### Логика работы

#### Создание напоминания (POST /api/reminders)
1. Публикуется `ReminderReceivedEvent` — логируется текст напоминания
2. Сохраняется напоминание в базе H2
3. Публикуется `ReminderCreatedEvent` (в транзакции) — добавляется "!" к тексту
4. Публикуется `ReminderNotifiedEvent` — асинхронно имитируется уведомление
5. Все шаги логируются

#### Обновление напоминания (PUT /api/reminders/{id})
1. Обновляется текст напоминания в базе
2. Публикуется `ReminderCreatedEvent` (без транзакции) — не обрабатывается транзакционным слушателем
3. Логируется публикация события
