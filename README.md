Проект позволяет создавать списки дел с названием и набором событий, а также получать их из бд
Данные храним в бд с использованием двух таблиц: `todo_lists` для списков и `events` для событий

## Особенности
- по умолчанию используется H2 база данных (в памяти)

## Как работает код

### Архитектура
1. **Контроллер (`TodoController`)**:
    - Обрабатывает HTTP-запросы (`POST /api/todos` и `GET /api/todos`)
    - Использует DTO (`TodoListDto`) для получения данных от клиента
    - Передает запросы в сервисный слой

2. **DTO (`TodoListDto`)**:
    - Record-класс с полями `name` и `events`
    - Содержит аннотации валидации (`@NotBlank`, `@NotEmpty`), чтобы гарантировать корректность входных данных

3. **Сервис (`TodoListService`)**:
    - Содержит бизнес-логику
    - Преобразует `TodoListDto` в сущности `TodoList` и `Event`, сохраняет их в базу через репозиторий
    - Извлекает данные из базы и возвращает их контроллеру

4. **Модели (`TodoList` и `Event`)**:
    - `TodoList`: представляет таблицу `todo_lists` с полями `id`, `name` и связью `@OneToMany` с `events`
    - `Event`: представляет таблицу `events` с полями `id`, `title` и связью `@ManyToOne` с `todoList`
    - Используются аннотации JPA для маппинга на таблицы базы данных

5. **Репозиторий (`TodoListRepository`)**:
    - Интерфейс, унаследованный от `JpaRepository`, предоставляет методы для работы с базой (сохранение, получение данных)

6. **Точка входа (`DemoApplication`)**:
    - Запускает Spring Boot приложение
    - Может содержать дополнительную логику

### Логика работы
#### Создание списка дел (`POST /api/todos`)
1. Клиент отправляет JSON:
   ```json
   {
     "name": "Мой список дел",
     "events": ["дело1", "delo2", "delo3"]
   }

2. `TodoController` принимает запрос, валидирует `TodoListDto` с помощью `@Valid`
3. Данные передаются в `TodoListService.createTodoList`
4. Сервис:
   - Создает объект `TodoList` с заданным name
   - Преобразует список строк events в объекты `Event`, связывая их с `TodoList`
   - Сохраняет `TodoList` в базу через `TodoListRepository.save()`, что автоматически сохраняет и связанные `Event` (благодаря cascade = CascadeType.ALL)
5. Возвращается сохраненный `TodoList` с сгенерированным id и связанными `Event`  

#### Создание списка дел (`POST /api/todos`)
1. Клиент отправляет GET-запрос на `/api/todos`
2. `TodoController` вызывает `TodoListService.getAllTodoLists`
3. Сервис вызывает `TodoListRepository.findAll()`, который извлекает все TodoList из базы вместе с их `events` (за счет связи `@OneToMany`)
4. Данные возвращаются клиенту в формате JSON
