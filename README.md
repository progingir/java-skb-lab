Тут есть метрики по колву запросов, времени их выполнения, колву купленных шоколадок и кастомный актуатор

## Что имеется
- **REST API**: для создания и получения информации о покупках
- **Метрики**:
    - подсчет общего колва запросов и по методам
    - измерение времени выполнения запросов
    - подсчет количества купленных шоколадок по типам
- **Кастомный актуатор**: логирует время вызова в консоль
- **Micrometer**: для сбора и публикации метрик
- логирование вызовов актуатора

## Как работает код

### Архитектура
1. **DemoApplication**:
    - основной класс

2. **PurchaseController**:
    - Эндпоинты:
        - `POST /api/v1/purchases` — создание новой покупки
        - `GET /api/v1/purchases` — получение списка всех покупок
    - использует `PurchaseService` для обработки запросов

3. **PurchaseService**:
    - управляет списком покупок
    - публикует метрики о количестве купленных шоколадок через `MeterRegistry`
    - преобразует DTO в сущности и обратно

4. **MetricsAspect**:
    - аспект для перехвата вызовов методов контроллера
    - собирает метрики:
        - количество вызовов методов (`app.requests.total` и `app.requests.<method>`)
        - время выполнения запросов (`app.timer.<method>`)

5. **LogTimeActuator**:
    - кастомный актуатор с эндпоинтом `/actuator/logtime`
    - при вызове логирует время и возвращает текстовое сообщение

6. **DTO и Entity**:
    - `PurchaseRequestDto`: record для входных данных 
    - `PurchaseResponseDto`: record для ответа
    - `Purchase`: сущность покупки
    - `Chocolate`: enum с типами шоколадок
