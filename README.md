Проект реализует REST API для управления списками задач с авторизацией и ролевой моделью

## Особенности
- Аутентификация через Basic Auth
- Ролевая модель: ADMIN, SUPPORT, USER
- Публичные и защищённые эндпоинты
- InMemory аутентификация (для демонстрации)
- Используется JSR-250 аннотация `@RolesAllowed`

## Как работает код

### Архитектура
1. **SecurityConfig**:
    - Настраивает доступ к эндпоинтам
    - `/public/api/**` - доступен всем
    - Остальные запросы требуют аутентификации
    - Включает поддержку JSR-250 аннотаций

2. **Контроллеры**:
    - `PublicApiController`: доступен без авторизации
    - `AdminApiController`: требует роли ADMIN
    - `SupportApiController`: требует роли SUPPORT

3. **Аутентификация**:
    - InMemoryUserDetailsManager с тремя пользователями:
        - admin (роль ADMIN)
        - support (роль SUPPORT)
        - user (роль USER)
    - Пароли хранятся в зашифрованном виде (BCrypt)

### Логика работы

#### Публичный эндпоинт (GET /public/api)
1. Доступен без авторизации
2. Возвращает строку "This is public API accessible to everyone"

#### Админский эндпоинт (GET /admin/api)
1. Требует авторизации с ролью ADMIN
2. Возвращает информацию об авторизованном пользователе

#### Эндпоинт поддержки (GET /support/api)
1. Требует авторизации с ролью SUPPORT
2. Возвращает информацию об авторизованном пользователе
