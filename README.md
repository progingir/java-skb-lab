### Аннотация *@SpringBootApplication*
Аннотация *@SpringBootApplication* является ключевой аннотацией в Spring Boot,
которая объединяет несколько других аннотаций для упрощения настройки приложения.
Она используется для маркировки главного класса приложения и активирует автоматическую
конфигурацию Spring, сканирование компонентов и другие функции.

*`@SpringBootApplication`* — это комбинация трех основных аннотаций:

-*@SpringBootConfiguration*  
-*@EnableAutoConfiguration*  
-*@ComponentScan*

### Параметры аннотации *`@SpringBootApplication`*

1. `excludeName`  
*Зачем нужен*: иногда автоматическая конфигурация
Spring Boot подключает ненужные или конфликтующие
компоненты, этот параметр помогает отключить конкретные
классы автоконфигурации, если их имена известны, чтобы избежать
нежелательного поведения


2. `scanBasePackageClasses`  
*Зачем нужен*: указывает классы, пакеты которых будут использоваться как
базовые для сканирования компонентов. Если нужно ограничить или явно указать,
где Spring должен искать компоненты, этот параметр
позволяет задать конкретные классы, начиная с их пакетов, 
это полезно для точной настройки области сканирования


3. `proxyBeanMethods`  
*Зачем нужен*: определяет, будут ли методы, помеченные как @Bean
в конфигурационном классе, создавать прокси, если установлено в true, Spring
создает прокси для обеспечения согласованности бинов, если false, прокси не создаются,
что может улучшить производительность в простых случаях, но требует осторожности
при управлении зависимостями


4. `scanBasePackages`  
*Зачем нужен*: указывает список пакетов, которые Spring будет сканировать для поиска компонентов
По умолчанию Spring сканирует пакет, где находится класс с @SpringBootApplication,
и все его подпакеты, этот параметр позволяет явно указать другие пакеты или расширить
область сканирования, если компоненты находятся в разных местах


5. `nameGenerator`  
*Зачем нужен*: позволяет указать пользовательский генератор имен для бинов,
которые создаются в процессе сканирования компонентов, если стандартные имена бинов 
не устраивают, этот параметр дает возможность задать собственную
логику генерации имен


6. `exclude`  
*Зачем нужен*: указывает классы автоконфигурации, которые нужно
исключить из автоматической настройки
